🎯 시나리오 1: 권한 상승을 통한 문화상품권 부정 취득

1️⃣ 정보 수집 (Reconnaissance)

공격자의 사용자 ID를 확인합니다.

2️⃣ 취약점 악용: 권한 체크 우회 (Broken Access Control)

관리자 권한이 없어도 포인트를 조작할 수 있는 취약한 API 발견!

3️⃣ CSRF 공격: 자동 문화상품권 교환

CSRF 보호가 비활성화된 상태에서 교환 API 자동 호출

4️⃣ XSS 공격: 악성 스크립트 게시글 생성

게시판에 악성 스크립트를 삽입하여 관리자 세션 탈취 가능

5️⃣ 전체 공격 체인 자동 실행

모든 단계를 순차적으로 실행합니다.

📚 공격 시나리오 요약

  1. 정보 수집: 자신의 User ID 확인
  2. 권한 상승: 관리자 권한 체크가 누락된 API 발견 및 악용
  3. 포인트 조작: 자신의 포인트를 999,999로 증가
  4. CSRF 공격: CSRF 보호가 없는 교환 API 호출
  5. 문화상품권 취득: 부정하게 획득한 포인트로 문화상품권 교환
  6. (선택) XSS: 게시판에 악성 스크립트 삽입하여 다른 사용자 공격

발견된 취약점

  • ❌ Broken Access Control (CWE-284): /admin/api/users/{userId}/points에서 관리자 권한 체크 누락
  • ❌ CSRF (CWE-352): insecure 프로필에서 CSRF 보호 비활성화
  • ❌ XSS (CWE-79): 게시판에서 HTML 이스케이프 없이 innerHTML 사용
  • ❌ Session Fixation (CWE-384): 로그인 시 세션 ID 재생성 없음
  • ❌ Missing Transaction: 포인트 차감 시 @Transactional 없음
